## 4. HSICS Classification Methodology

### 4.1 CRITICAL FINDING: Inconsistent Industry Mapping

The report (Section 3.3, lines 162-170) describes a 7-category code-range industry classification. However, the three Python scripts that implement industry neutralization each use DIFFERENT mappings:

| Script | Categories | Report Match? |
|--------|-----------|---------------|
| `industry_neutral_analysis.py` (line 41-58) | **16** categories | NO -- report says 7 |
| `neutral_excess_analysis.py` (line 102-111) | **8** categories | NO -- different boundaries |
| `hsics_final.py` (line 64-79) | **19** ranges (fallback) | NO -- HSICS-based, not code-range |

**Specific mismatches for `industry_neutral_analysis.py` (the script that generates plots used in Sections 7 and 13):**

The report claims these 7 ranges:
```
00001-00099: 金融/公用/综合
00100-00999: 地产/建筑/工业
01000-01999: 消费/零售/医疗
02000-02999: 综合企业
06000-06999: 银行
07000-07999: 科技/互联网
08000-09999: 中小型股
```

The actual code in `industry_neutral_analysis.py` uses these 16 ranges:
```
1-29: Financials
30-99: Utilities_Conglomerates
100-299: Property
300-499: Industrial_Manufacturing
500-999: Consumer_Industrial_Misc
1000-1499: Consumer_Retail
1500-1999: Industrial
2000-2499: Technology
2500-2999: Consumer_Discretionary
3000-3499: Healthcare
3500-3999: Biotech_Pharma
4000-4999: Others
6000-6999: Financials
7000-7999: Technology
8000-8999: Small_Cap_Misc
9000-9999: Small_Cap_Others
```

Key differences:
- Code has `3000-3499: Healthcare` and `3500-3999: Biotech_Pharma` -- report does NOT mention these at all
- Code splits `1-99` into `1-29` and `30-99` -- report lumps them together
- Code has far more granular ranges (16 vs 7) -- results may differ from what the report implies

## 6. Summary of Issues by Severity

### HIGH SEVERITY

1. **Industry mapping inconsistency (Section 4.1 above)**: The report's description of the "code-range industry classification" (Section 3.3, lines 162-170) does NOT match the actual code in `industry_neutral_analysis.py` (lines 41-58). The report says 7 categories; the code uses 16. The code also includes ranges (3000-3999, 4000-4999) not mentioned in the report. This means the reader cannot reproduce the reported results from the methodology described.

### MEDIUM SEVERITY

2. **Missing Beta values in sensitivity tables (Section 1.4 above)**: Lines 896-925 of the report show "---" for Beta at n=10, 15, 30, 40. The source script (`te_analysis_plots.py`) only computes Beta for n=20 and n=50 (lines 138-145). All n should have Beta computed for a complete table.

3. **HSICS mapping methodology not documented (Section 4.2 above)**: The report states HSICS is used but never explains the two-tier mapping process (akshare JSON + code-range fallback). A reader cannot understand or reproduce the classification.

### LOW SEVERITY

4. **12+ unreferenced plot files (Section 1.3 above)**: Multiple scripts generate plots that are not included in the report, creating confusion about which plots are "official" and wasting disk space.

5. **Orphan file `v4_plot_08.png`**: Exists in `pict/` but has no corresponding Python script or report reference. Likely a leftover from a deleted analysis.

6. **Redundant scripts**: `hsics_final.py` and `industry_size_neutral.py` both perform HSICS-based industry+size neutralization. `v4_industry_neutral.py` and `industry_neutral_analysis.py` both handle crude industry neutralization. Consolidation would reduce maintenance burden.
